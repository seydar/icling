Examine and Change CFGs

  • Look at a Cfg (5 min) 
              >> cd mxcom/m
          >> type E.cfg
  • Using pencil and paper, use E.cfg to parse i*i eof. (5 min) (answer)
  • Using pencil and paper, use E.cfg to parse (i) eof. (5 min)
  • Make up a few more examples. (10 min)
  • Using pencil and paper, use E.cfg to detect an error in i**i eof (5 min)
  • Look at Cfg tables for E.cfg (5 min) 
              >> c = Cfg(xread('E.cfg'))
          >> c.Vi
          >> c.Vn
          >> c.V
          >> c.dump()
    Note that in V, the input symbols are first, phrase names next. This is a (traditional) hack so that it is easy to tell them apart.
  • Change the first rule to 
              E
          S eof
    denoting a sum, then modify the rest of the grammar to accomodate +. Repeat the first part of this exercise to see if Cfg.m can handle your changes. (answer) (10 min)
  • Continue, adding - and / for subtract and divide. Again, see if Cfg.m can handle your changes. (10 min)
Extra credit
  • Continue, adding D after i, and define D to be 0,1,2,...,9. Again, see if Cfg.m can handle your changes.
  • Peek at the LR(1) matrix. We will talk about how to make and use it later.
